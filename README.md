# 三餐随机选择助手

一个帮助你决定每天吃什么的三餐随机选择Web应用。

## 功能特点

- 🍳 **三餐随机选择**：早餐、午餐、晚餐一键随机选择
- 📝 **食物管理**：添加、编辑、删除食物，支持标签分类
- 🏷️ **标签筛选**：按标签快速筛选食物
- 💾 **本地存储**：数据自动保存到浏览器本地存储
- 📱 **响应式设计**：支持手机和电脑访问

## 使用方法

### 本地运行

1. 下载所有文件到本地目录
2. 直接用浏览器打开 `index.html` 文件即可使用

### 部署到 GitHub Pages

1. 在 GitHub 上创建一个新仓库
2. 将所有文件上传到仓库
3. 进入仓库 Settings → Pages
4. 选择 Source 为 `main` 分支（或 `master` 分支）
5. 点击 Save，等待几分钟后即可访问

访问地址格式：`https://你的用户名.github.io/仓库名/`

### 部署到其他静态网站托管服务

#### Vercel
1. 安装 Vercel CLI：`npm i -g vercel`
2. 在项目目录运行：`vercel`
3. 按照提示完成部署

#### Netlify
1. 访问 [Netlify](https://www.netlify.com/)
2. 拖拽项目文件夹到 Netlify 部署区域
3. 自动完成部署

## 使用说明

### 随机选择三餐

- 点击"一键选择三餐"按钮，自动为三餐各选择一个食物
- 或分别点击早餐、午餐、晚餐的"开始随机"按钮
- 点击"清空"按钮可以清空对应餐食的选择

### 管理食物列表

- 点击右下角的"+"按钮添加新食物
- 在食物卡片上点击"编辑"按钮修改食物信息
- 点击"删除"按钮删除食物

### 标签筛选

- 点击标签栏中的标签可以筛选对应标签的食物
- 点击"全部"标签显示所有食物

### 添加标签

- 在添加/编辑食物弹窗中，可以输入自定义标签并添加
- 标签会自动保存，下次添加食物时可以使用

## 技术说明

- **纯前端实现**：无需后端服务器，所有数据存储在浏览器本地
- **原生JavaScript**：不依赖任何框架，代码简洁易懂
- **响应式布局**：适配手机和电脑屏幕
- **本地存储**：使用 localStorage 保存数据，刷新页面不丢失

## 浏览器兼容性

- Chrome/Edge（推荐）
- Firefox
- Safari
- 移动端浏览器

## 注意事项

- 数据存储在浏览器本地，清除浏览器数据会丢失所有记录
- 建议定期备份重要数据
- 如需在不同设备间同步数据，可以考虑导出/导入功能（未来版本）
